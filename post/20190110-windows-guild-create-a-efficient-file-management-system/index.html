Windows 教程3-基础阶段-打造一个高效的文件管理系统

2019年01月10日
 read
   Categories: 软件技巧

经过Windows 教程1-如何下载官方 ISO及Windows 教程2-如何安装 Windows 系统我们已经将 「Windows 系统」安装到「C 盘」，后续的步骤无非就是「分盘、安装相关软件、系统调优」！

分盘是后续步骤的第一步：「分盘」，后续所有的操作都建立于此，但是如果仅仅是「分盘」我就没要单独成篇，我在本文想要做的告诉大家如何搭建一个「高效的文件管理系统」！

本文大部分内容来自易道非博客-知识管理系统——知识存储及发布端 V1.0 版，主要区别如下：

  1. 本教程的受众主要是非计算机专业人士/爱好者，所以相对应的做了部分的简化
  2. 补充相关的文件管理效率软件，如 「Everything」等

最完美的分盘方案

不知道大家的硬盘是怎样分盘的，是一台电脑就一个C盘，还是说分成C、D两个盘……

我将我的分盘方案列出并说明为什么如何分盘？

盘名 大小 原因 追加
C 盘 50 G 1. 系统盘分离可防御如系统崩溃重装导致资料丢失风险 2. Windows系统安装完成一般为35 G 最大60G
D 盘 80 G 1. 公共资料盘，主要用于存放程序安装文件、公共资料 2. 系统重装部分软件无需重新配置，节省时间
E 盘 80 G 1. 私人资料盘，主要用于存放个人资料——利用 windows 自身的 「BitLocker」加密 私人密码
空白 27.22 G 1. 用于后期备用

以上分区可以说是我使用电脑以来做出的最完美的分区方案——系统崩溃风险、公共使用、个人隐私三者全部兼顾！

在书写易道非博客-知识管理系统——知识存储及发布端 V1.0 版时，我曾经发布过2个任务：

任务1：以上方案针对的情景是哪种情景？是否适用工作情景

任务2：请思考如不新分盘如何进一步加强隐私？如无任何规定如何进一步加强隐私？

下面开始公布答案：

任务1 答案
以上的方案如果仅是针对私人领域，那完全没问题，但是如果要兼顾办公领域（即电脑私人与工作文件融合在一起）,那么我们的文件系统需要增加以下因素：

  1. 当前工作文件——除同事及本人外其他人不可访问
  2. 增加参与人员：同事等工作人员

所以针对性的需要创建一个「工作盘」，并利用 Windows 自身的「BitLocker」创建一个工作密码，保证仅有自己与同事等工作人员可访问！

任务2 答案
利用VeraCrypt软件创建某个加密空间，将相关资料放入其中即可

此方式方法可用于某些重要文件整体加密打包上传至云端网盘！

恰到好处的文件夹分类

同样的问题

  1. 不知道大家的文件是怎样分，是电影一个文件夹、音乐一个文件夹还是……
  2. 如果不借助搜索你能够在打开几层夹就能找到你要的文件

参照《强力底版的多领域实践 | 知行，在路上》的「六位平衡术」以及《如何构建自己的笔记系统？ - Lachel的回答》「知识内化需经过：收集——完善——整理——（应用）——归档」将文件夹系统调整如下：

内在原则如下（Copy 自《如何构建自己的笔记系统？ - Lachel的回答》 ）：

1.一切以「用」为主。记笔记是为了用，不用的笔记没有价值

2.以「主题」为核心。「主题」即「项目」，也就是「笔记可能的用途」。记笔记的目的，在于支撑主题，一切流程都是为主题服务。

类似的可以将 RSS/浏览器书签等进行相关的分类！

规范的文件命名

参考个人文档命名规范和文件夹分类以及拯救你的电脑之文件命名规范与目录规划 以及工作经验整理如下「文件命名规范」：

文件主题+版本号+相关部门+文件作者+相关日期

eg:知识管理系统之知识存储及发布端 V1.0-研发部-易道非-20181219

关于其中的版本号可以采用采用不同的粒度控制，如 V1.0.0、V1.0.0.0等！

以防万一的网络云备份系统

不知道有多少人有关键文件备份的习惯，反正我是一定要做的！

因为我清楚的记得2016年我自己一块西部数据的硬盘忽然挂掉时的感觉——「fu**ck，这种事情怎么会发生」！ 同时在网络上也可以随时找到类似的「数据丢失，损失惨重」的案例：

  1. 静默错误：为什么看了那么多灾难，还是过不好备份这一关？：静默错误导致「前沿数控」2年核心数据丢失，由于无备份公司直接从0开始
  2. 传华为被罚5亿，因数据丢失事件遭中国移动“全国系统排查”：数据误删导致广西80万移动用户2G/3G/4G业务无法使用
  3. 下厨房6月26日数据丢失事故总结
  4. ……

构建网络存储体系大致分为以下2步：

  1. 文件分类：按照来源、恢复难度、价值大小、隐私性分类
  2. 针对文件分类选用合理的存储方案存储

由于本文针对的是「非计算机专业人士/爱好者」所以将原来的方案初步完成的网络存储体系调整如下：

文件类型 云盘 :–
软件、电影、音乐等无隐私不重要的文件 百度云
重要但保密性低私人文件 百度云
重要但保密性高且变动少使用率低的私人文件 利用加密软件上传至百度云
重要但保密性高且变动多或使用率高的私人文件 利用加密软件上传至坚果云 速度、增量同步
重要但保密性低工作文件 坚果云
重要但保密性高工作文件 利用加密软件上传至坚果云

PS：
以上加密软件分别为「VeraCrypt」与「Cryptomator」，两者之间的区别是：

「VeraCrypt」采用的是将待加密文件整体打包加密后上传，所以比较适合比较文件关联性强文件比较小或文件使用频率极低的情况 「Cryptomator」采用的是将单个文件本地加密后上传，所以比较适合文件关联性不大或文件比较大的情况

文件管理效率软件

文件管理效率软件大致有

  1. 文件管理器：代替 windows 文件管理器，「Total Commander」、「FreeCommander」，学习成本比较高，不推荐使用
  2. 文件名查找工具：「Listary」、「Everything」
  3. 全文搜索工具：「docfetcher」、「FileLocator」

20190111 搜集到的相关教程如下：

  1. 善用佳软-最易读的Total Commander教程
  2. 一本关于 Total Commander 的书
  3. Listary 官方文档中文版
  4. 善用佳软-Everything:速度最快的文件名搜索工具

个人建议整体方案是 「Listary」 +「Docfetcher」

  • 「Listary」：介于搜索工具、快速启动工具和文件资源管理器之间的软件，通过双击 「CTRL」 键弹出操作界面，可快速检索文件名、调用程序等
  • 「Docfetcher」：Java 跨平台全文检索工具，建立完成索引后能够快速定位文件内部内容
