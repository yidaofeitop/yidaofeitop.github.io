虚构互联网形象

互联网时代，保护自己的个人隐私：基础篇我已经讲过为什么要保护我们的个人隐私，但是一直都没有说明 信息泄露之后别人是如何「定位」你的，借助「虚构互联网形象」章节从「社会工程学」角度简单的说明:

身份隔离

本文梳理自：如何隐藏你的踪迹……从【身份隔离】谈谈社会工程学的防范，内容如下：

我们每注册一个网络账号基本上都会提供如「手机号码、注册地址」等相关的信息，这些可以称之为属性，而如「骇客」、「黑帽子」等就能够通过这些属性来定位你

  • 如果知道你的「性别」，那么就能够将范围缩小到二分之一
  • 如果知道你的「星座」，那么就能够把范围缩小到十二分之一。
  • 如果知道你的「身份证号」，那么就能够基本上可以唯一地锁定此人
  • ……

这就是为什么我建议大家树立起「信息分级/多主键、最小化原则」的原因，因为只有这样才能够让所有想要「破解你的人」无法画出你完整的画像！

这里需要多提一句的是「多主键」，能够作为「主键」我按照一般的梳理如下：

  • 明面个人属性：身份证号、手机号、邮箱地址、家庭住址
  • 潜在硬件属性：手机 IMEI 码、网卡 MAC 地址、公网IP
  • 潜在软件属性：浏览器指纹技术、Cookie 技术
  • 潜在个人属性：个人习惯

针对性的措施如下：

  1. 信息分级/多主键
  2. 最小化原则
  3. 尽量使用高隐私软件（国内与国外、商业组织与非盈利组织、开源与闭源……）
  4. 使用各种技术手段屏蔽隐私泄露风险，如邮件客户端使用「雷鸟」、关键数据使用加密软件「TrueCrypt」、「BitLocker」加密

此部分措施具体细节可以访问互联网时代，保护自己的个人隐私：基础篇！

另外针对「匿名身份」应该建立一套独立的与其他账号系统无任何交叉的体系，具体的措施如下：

  1. 建立「专属」的系统-虚拟机/独立硬件/多重代理
  2. 专有的 TOR 通道
  3. 相关账号不绑定手机或这使用境外不记名手机卡
  4. 不要在手机上操作相关帐号或者直接使用功能机

个人习惯

之所以独立出来，我觉得是因为这个此问题是一个比较难以深入的但是又很重要的问题，举一个最简单的例子：文章风格，每个人的行文风格都是有差异的，两篇发表在不同平台的文章如果行文风格相同基本可以认定这两个作者很有可能是同一个人，如果进一步分析两者有相互引用那基本就可以判定两者是同一个人，这完全是有可能做到的！

下面就梳理以下哪些个人习惯可能会导致个人隐私的泄露：

  1. 行文风格，基本可以类比笔迹
  2. 不同账号体系之下文章关联程度：文章细节摘抄、相互引用权重
  3. 不同账户活跃程度高度相同
  4. 网络活跃时间规律
  5. 选用多种方式与其他人沟通

针对性的措施初步构想如下：

  1. 不同账号体系之下尽量做到无关联性，甚至将相关的主题完全分割，这样就可以避免相互引用，内容直接部分 Copy
  2. 人为的制造随机「静默期」
  3. 沟通方式就以评论区/邮件为主阵地

破解硬件风险

以上基本解决除去「潜在硬件属性」之外的个人隐私泄露的渠道，本章节就主要处理如何解决「潜在硬件属性」，其中以「隐匿公网IP」方案最为复杂所以先介绍此内容：

隐匿公网IP

首先大家需要准确理解「公网 IP/内网 IP/NAT 协议/ DHCP 技术」的概念，如不理解请直接访问知乎-知乎用户-公网IP会被用完吗？

针对公网IP的防范手段仅有2种：

  1. 多重代理+虚拟机
  2. 公共场所上网

关于后者不予多讲，主要以前者为主题展开！

隐匿公网IP-多重代理

先声明一下：本文所说的「代理」是广义的，包括了：「传统意义上的代理」和「VPN」等。

平时咱们用代理来翻墙，大部分属于一重代理——不论你用的是 VPN 还是 HTTP Proxy 还是 SOCKS Proxy，当中都只有一个服务器进行中转

而多重代理的概念则是多重服务器进行中转，具体如下：

如果按照一般的推理是以两个翻墙工具组成形成二级代理，但实际上有一种非常好的方案「一重代理+Tor 浏览器」

单/多虚拟机

如果仅仅是使用代理的方式，你也面临「Flash、某些软件」都有一定的机制暴露公网 IP(具体见如何隐藏你的踪迹……[6]：用虚拟机隐匿公网IP（原理介绍）)

针对性的提出单/多虚拟机模式

  1. 单虚拟机
    我们把那些有危险的软件安装在虚拟系统（虚拟机A）内，然后把该虚拟唯一的虚拟网卡设置为 「Host-Only 模式」，由于唯一的虚拟网卡是 Host-Only 模式，所以 虚拟机（Guest OS）内的任何软件都 不可能直接访问外部网络

  2. 多虚拟机
    基于以下原因：

    • 某些翻墙代理软件不是开源的，有些网友对这类软件不放心
    • 某些网友的 Host OS 不是 Windows，但是很多翻墙代理是 Windows 软件

    　　所以我们可以在「单虚拟机」的基础上再扩展一下，把代理软件也放到虚拟机中（下图的虚拟机B）。 虚拟机B 的网卡很有讲究滴——必须是双网卡。一个网卡设置为「 Host-Only 模式」，以便跟虚拟机 A对接；另一个网卡设置为 「NAT 模式」，以便访问外网。

其他硬件防护

  1. 修改手机 IMEI 码
    具体请参考小米系列手机修改IMEI码详细教程，但是就像我在互联网时代，保护自己的个人隐私：基础篇里讲的一样「移动端相比 PC 端，由于资源等的限制可操作空间非常少」，所以不建议大家没必要做如此「硬件」上的修改
