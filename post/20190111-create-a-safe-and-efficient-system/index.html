病毒定义、特性、分类

首先还是先定义什么是病毒吧！

按照1994年《中华人们共和国计算机安全保护条例》病毒的定义为：

计算机病毒是指编制、或者在计算机程序中插入的，破坏计算机功能或数据、影响计算机使用，并能自我 复制的一组计算机指令或者程序代码

病毒的主要特性：感染性、潜伏性、可触发性、破坏性、衍生性，关于其中的定义具体请查阅《计算机病毒原理及防治……》第7-13页

计算机病毒从不同的角度可以分解为不同的类型（如下表），这里必须提一句的是我们说的病毒一般都是指 「Windows 病毒」，至于为什么主要是因为：

  1. 整体框架上 Linux 的安全机制是远好于 Windows 的安全机制
  2. Linux 开源系统，Windows 闭源系统，从发现 Bug 的几率来讲前者就远远超出后者，还不考虑 Linux 主要用户都是 IT 相关人员
  3. ……

病毒工作原理以及入侵渠道

病毒的主要来源详细大家都有一定的了解，就是网络、外部存储介质（移动硬盘、优盘、光盘……）！

那么病毒到底是如何运作的呢？

首先我先大体说一下病毒的工作原理：

计算机系统的内存是一个非常重要的资源，我们可以认为所有的工作都需要在内存中运行（相当与人的大脑），所以控制了内存就相当于控制了人的大脑

病毒一般都是通过各种方式把自己植入内存，获取系统最高控制权，然后感染在内存中运行的程序。（注意，所有的程序都在内存中运行，也就是说，在感染了病毒后，你所有运行过的程序都有可能被传染上，感染那些文件这由病毒的特性所决定）

下面分别以常见的「程序型病毒」说明病毒的内部工作原理：

他是目前最为广泛的一种病毒，主要感染的 「.exe」 和 「.dll」 等可执行文件和动态连接库文件

运作主要分为2个阶段（以下部分引用自CSDN-langhaixin-计算机病毒运行机制）：

  1. 驻留（初始化）部分

    当运行了带有病毒的宿主文件 「.exe」 或是 「.com」 更可能是现在流行的「.vbs」的脚本文件，即激活了病毒的驻留程序。一般驻留程序会占据宿主文件的一部分，因此驻留程序首先会恢复这一部分的宿主程序；其次可能会判断系统内存是否已驻留了病毒，如果已驻留了则转去执行宿主程序，没有则将病毒程序驻留到内存中，驻留完成后再转去执行原宿主程序。

    大部分的病毒的驻留（初始化）部分都采用的这种模式，差别就在各种病毒的实现方法不同而已，DIR病毒首先修改该执行文件目录项中的首镞，首先调入内存的是位于磁盘尾镞的病毒程序，即激活了该病毒的驻留程序。

  2. 传染传播部分

    这个部分是病毒运行的一个核心部分，计算机病毒就像人体病毒一样会传染，无法感染其他细胞的病毒，总会死在白血球抗体的手上，同样无法感染其他文件的计算机病毒也是无法生存的。在攻击传染这部分所使用的手段也是各有各的不同，但其主要还是实现象没有染毒的文件传染病毒这一目标。

    有些病毒通过拦截中断向量，对申请中断的文件一一检测，如没有感染病毒则对其进行感染再去执行原中断，如是带毒文件的话则直接执行原中断。采用这种手段的有Yankee病毒，Dir病毒还有臭名昭著的CIH。

    还有的病毒则是对磁盘上所有可感染的文件进行扫描，对其中没有感染的文件进行感染。最近十分流行的Nimda病毒在单机上就是采用这种感染方式的。

    随着网络的迅速发展越来越多的病毒开始通过网络传播，常见的蠕虫病毒是通过扫描Microsoft Outlook系列或是其他邮件软件的通讯簿选取其中的几位或是所有人的邮箱进行发送以染毒文件为附件的邮件。或是针对各种服务器的系统Bug进行感染传播。

病毒防御手段

上面已经将计算机病毒的基础知识做了基本的介绍，相信大家对于计算机病毒有了一定的了解，那么现在我们来聊一聊如何防御病毒！

不知道上面提的问题「问题2：大家防治计算机病毒的手段大致有几种？」大家有几种答案？

我将我的答案列举出来，并续针对性的告诉大家为什么要如此做：

  1. 系统权限分级-新建一个「普通用户账户」做常用的用户
    管理员可操作系统所有的资源，而建立一个「普通用户」能够让程序进行某些高风险操作时必须经过管理员同意，这样就可以拦截一部分的病毒与高危操作（如修改注册表……）
  2. 系统杀毒软件-火绒/360国际版
    • 前者相比如「金山毒霸」等国内其他杀毒软件调优软件更为简洁干净，在杀毒的基础上将最真实的需求如「家长控制-上网时间」等糅合在一起，是在安全与系统管理软件里目前最佳的，所以本人将其作为常用软件， 关于此进一步可以访问《Windows 教程 4 - 基础阶段 - 好的软件改变你的生活-安全类-火绒》
    • 后者的「病毒查杀能力」较前者强，但是由于本人对「360 软件」一贯不信赖仅偶尔使用
  3. 可疑程序采用「虚拟机」或「沙盒程序」运行
    • Why？ 请访问：知乎-KaiwenSun-病毒代码需要debug吗，测试的时候难道不会损坏本地设备吗？
    • 前者要优于后者，具体请访问：如何保护隐私[9]：如何限制桌面软件的流氓行为？-为啥“OS虚拟机”的隔离性比“沙箱”更好？

个人信息自我防护

个人信息自我防护大致分为以下步骤：

第一步：信息分级：

看了很多网上关于个人信息防护的资料，发现基本上都是聚焦在如何利用某些工具防止泄密，很少有人谈到信息分级的问题！

为什么要如此做呢？

因为我们随着时间的推移，我们与外界的交互越来越多的时候就会产生非常大量的「连接」，但是这些「连结」相对你来说有重要的不重要的等情况，你如果将其一视同仁的话势必会造成管理成本上的上升，所以我们要将其分层安排不同的精力去管理！

我建议大家将信息分为三层：

  1. 非常重要，一旦泄露被恶意使用就会造成巨大损失
  2. 重要，一旦泄露会造成不是很重大的损失或会造成一定的困扰
  3. 普通/无所谓，泄露无任何损失

给大家举一个例子：

  • 支付宝支付密码泄露可能将造成个人财产的重大损失——非常重要信息
  • 淘宝登录账号泄露可能造成个人联系方式、购买记录、家庭地址泄露——重要信息
  • Adobe 账号泄露无任何损失——普通/无所谓信息

第二步：建立管理系统：

如果说管理系统也不是很正确，称之为建立不同的策略更为恰当，下面我将自己使用的策略列举出来：

  1. 手机号码分级-划分为非常重要、重要、普通/无所谓，推荐软件 「阿里小号」
  2. 邮件分级-划分为非常重要、重要、普通/无所谓，推荐大家临时邮箱
  3. 个人信息上传问自己以下问题：
    • 网站数据能否注销
    • 这个数据是否一定要上传、此类服务是否有可替代服务
    • 数据是否做了相关的加密或水印（尤其是证件类）
