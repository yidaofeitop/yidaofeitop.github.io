2019年01月15日
 read

翻墙如果选用自建翻墙渠道那么第一步就是选用合适的 「国外 VPS」，本文就是教大家如何完成这一步！

本文主要内容如下：

  1. 主流 VPS 服务商
  2. VPS 基础配置以及相关工具

主流 VPS 服务商

主流的 VPS 服务商按照国内外大致有以下几家：

  • 国外
    • 侧重小型 Iaas 提供商：bandwagong、vultr、Linode、DigitalOcean
    • 侧重 Saas 和 Paas 提供商：亚马逊 AWS、微软 Azure和谷歌 GCP
  • 国内
    • 阿里云、腾讯云、百度云、华为云

因为是翻墙所以其 VPS 的主机必须选用位于国外的，以性价比及隐私考虑出发建议大家选用「bandwagong、vultr、Linode、DigitalOcean」四家之中的任意一家即可！

关于以上四家 VPS 测评的文章网络很多，加上大家的需求不一样，就不一一详说！

PS：由于国内屏蔽，为节省大家时间特意列明国内相关的推广网站，大家可以从以上入口进入对应的官网（特此申明无任何利益关联）：

VPS 基础配置以及相关工具

基础工具

在使用 VPS 的过程中大致需要使用以下工具：

  1. Xshell-360：用于 SSH 连接 VPS 远程控制
  2. IP 是否被封测试：https://tools.ipip.net/ping.php
  3. 国内测试-端口是否被封测试：http://tool.chinaz.com/port/
  4. 国外测试-端口是否被封测试：https://www.yougetsignal.com/tools/open-ports/

「2-4」项主要用于检测「VPS」是否被 GFW 封锁，具体使用方法是：

  1. 第二项，亦可使用：控制台（CMD 终端或 Linux 终端）输入 ping xxxx.xxxx.xxxx.xxxx 测试，看丢包率即可
  2. 第三项，如果出现 「Close 或关闭」 则需要进行下一项操作
  3. 第四项
    • 如果第二步骤为 「Close」，而本步为 「Open」，则基本确定被封
    • 如果第二步骤为 「Close」，而本步为 「Close」，则需要排查防火墙配置

VPS 基础配置

下文以 bandwagong 为主讲解：

第一步 购买 VPS

如果纯粹作为翻墙使用建议购买 bandwagon
具体的入口可以可以参考搬瓦工 VPS ,选择最便宜的年 $ 19.99 方案（ KVM 架构）即可

第二步 进入 kiwivm 控制台

进入 VPS 管理页面
「MyService」->服务器实例->「kiwivm Control Panel」

第三步 选用操作系统

「Install new OS」->选择相关的系统（建议选用 「Centos」），安装完成后点击「Main controls」，复制此处的 IP 地址与 SSH 端口

另外如在操作系统安装完成后未记录「root 密码」请点击「Root password modification」->「Generate and set new root password」获得密码

本操作完成后 VPS 基本能够获得 VPS 的 「IP地址」、「SSH 端口」、「root 密码」

第四步 XShell 远程操作 打开 「XShell」软件->点击「新建连接」，输入上一步骤获取的 VPS 的 「IP 地址」、「SSH 端口」

后续会弹出「密码输入框」，输入上一步骤的密码即可！

远程登录后第一步请修改「root 密码」，具体的命令如下：

$root>passwd   
$root>输入新密码（不会以明文显示），回车  
$root>再次输入新密码  
$root>提示修改成功

后续的安装等步骤请访问具体的翻墙技术文章！

其他 VPS 的操作文章也可以通过相关的推广网站获得最新的操作指南！
