我们生活在一个怎样的时代

以下内容引用自浅谈互联网隐私安全

上世纪90年代，《纽约客》曾有一句俚语闻名全球：On the Internet, nobody knows you’re a dog（在互联网上，没有人知道你是一条狗）。也正是从那时候起，互联网开始兴起，其不但促进信息流动效率，而且以虚拟和高度匿名的优势吸引大批用户，任何用户都可以在网络上打造一个全新的完美的无懈可击的自我，屌丝瞬间逆袭成高富帅。似乎那时对匿名性的痴迷胜过了当今的美颜，在各种社交软件上以各种各样的目的侃侃而谈着不知从哪编造的各种经历。那时候，互联网在人们心里的功能可能就是：玩（玩游戏）、聊（聊天交友）、约（此处过滤）、骂（吐槽）。

进入21世纪以来，我国互联网快速发展，尤其是移动互联网进一步推动这种速度，很快我们发现，如今的互联网已经与当初的截然不同，电商购物、地图导航、移动支付、炒股理财、网络约车、政务办公、充值缴费等等，似乎一切都能在互联网上完成，出门不需要再带钱包，全靠一部手机即可。

但是这一切的背后也带来了很多问题，尤其在移动互联网时代似乎没有绝对意义上的隐私。很多产品为了用户体验搜集用户使用信息，而这些信息有时并不能明确鉴定是否属于个人隐私。原因在于，互联网的出现使得很多隐私不断公开化，而使得隐私失去了私密属性。当我们的账号密码被挂在“社工库”中，当我们接到假冒卖家打来的电话时，当我们手机每天收到“借款放贷、投资理财”的骚扰电话时，隐私似乎已经离我们越来越远。

“不要和陌生人说话”这句话想必大家都非常熟悉。为了保护自身安全，我们用简单粗暴的方法将社会群体划分为“熟人”和“陌生人”，而这两者则对应“安全”和“非安全”两种状态。在保护隐私方面，一般坚持最小化原则，即让尽可能少的人知道，最理想的状态是，不让与该隐私无关的人员知道任何信息。

但是互联网的生态环境完全改变了这一状态，有时隐私保护的主动权似乎不在你手中。由于之前网络的高度匿名带来的很多不好的现象，目前推出的各种网络实名制需要用户或多过少的提供个人信息进行身份认证，比如与金融理财相关的应用实名认证要求最为严格；另外，一些产品为了提升用户体验需要搜集一些个人信息，比如浏览记录等。而当“大数据”被炒的很热的时候，各大公司也越来越重视“数据”的重要性，个人数据则起到重要作用。就这样，有意无意中，原本属于个人的“数据”源源不断的输向云端，而云端的一切对于用户来说都是陌生的，因此之后所发生的各种问题，用户也不得而知，也不可控制。

在这样一个「IOT」和「大数据」的时代，我们每个个体无时无刻的产生着各种数据，而这些数据被以不同形式收集记录着，每个人似乎变得更加透明，如果有人拿到这些数据基本上就可以画出你的人物画像，从而针对你做各种的营销……

这是一个不可逆转的潮流，并且会越来越快！我们能够做的就是2个：

  1. 呼吁国家尽快完善相关的立法-《个人信息与隐私保护法律法规现状》报告
  2. 个人做好相关的个人隐私保护

Android 系统

以下内容抽离自知乎-Yogi

Android 的本质是 Linux，而基于 Linux 的「沙盒安全机制」，理论上讲，在没有获取到最高权限 ROOT 时，整个系统都是安全——在没有 ROOT 的情况下，沙盒机制能让程序间彼此独立，从而保护程序数据。

打个比方，如果说病毒木马是一头饿狼，网银等其他程序是羔羊，那么 「ROOT机制」能将饿狼和羔羊分开隔离于各自的铁笼内，这时候，羔羊是安全的。一旦破了 ROOT，这个铁笼也就打开，饿狼就可以肆意攻击了。

当然如果你有极致的要求，推荐你访问一下：隐私工具……手机操作系统

由于国内的 Andriod 系统已经基本集成相关的安全软件以及流量统计等功能，所以大家要做的就是利用相关的权限去控制相关软件 的行为，如禁止某个应用上网等

防范软件层面隐私泄露

PC 端

浏览器

推荐使用 Chrome/Firefox 浏览器并进行如下的设置：

  • 隐私安全：开启DNT（Do Not Track），禁止第三方Cookies
  • 隐私安全：关闭记住密码功能或者谨慎使用此功能
    • 官方-如何关闭记住密码功能
    • 知乎-hell0w0rld-Chrome-密码采用明文存储机制：多注意看评论
  • 安装插件「AdGuard」、「Ghostery」、「HTTPS Everywhere」、「Browser Plugs-指纹隐私防火墙」、「Self-Destructing Cookies-自动删除 Cookies」
  • 使用注重隐私的搜索引擎，如「duckduckgo」

其他软件

按照如何保护隐私[1]：如何选择软件和服务？所说挑选软件需要遵循以下原则：

  • 能用纯 Web 搞定的，尽量不要装软件
  • 尽量用非营利机构的软件和服务（其中 「Netflix」公司通过大数据购买《纸牌屋》版权可以作为大数据商业应用案例之一）
  • 尽量使用国外的软件或服务，因为就算隐私泄露 1、他们也无司法管辖权；2、关注点不一样
  • 尽量使用「开源软件」以防止后门和偷窥隐私
  • 如果软件本身的用途跟网络无关，那就尽量选「单机软件」
  • 要避免使用高权限用户，技巧之一就是：尽量用绿色软件

移动端

移动端相比PC端，由于资源等的限制可操作空间非常少，我针对相关的可能泄露的点整理如下：

  1. 尽量关闭手机系统自身的关键云同步，尤其是短信云同步，具体案例见知乎-卡瓦伊-破解小米云从而破解支付宝安全体系
  2. 使用「Google Play」、「酷市场」、「官方网站」等正规可信赖渠道下载相关应用
  3. 高风险应用预先使用「逍遥安卓」类「Andirod 模拟器」运行判断
  4. 应用安装完成后注意使用系统自身的权限管理系统设置相关的权限，如无必要可将其网络权限关闭，需要时再开启
  5. 关键隐私应用请全部设置指纹解锁等

防范互联网隐私泄露

整体原则

我防范隐私泄露的整体原则是信息分级、多主键、最小化原则，具体说明如下：

信息分级、多主键

我的方式方法是将信息分为三层：

  1. 非常重要，一旦泄露被恶意使用就会造成巨大损失
  2. 重要，一旦泄露会造成不是很重大的损失或会造成一定的困扰
  3. 普通/无所谓，泄露无任何损失

给大家举一个例子：

  • 支付宝支付密码泄露可能将造成个人财产的重大损失——非常重要信息
  • 淘宝登录账号泄露可能造成个人联系方式、购买记录、家庭地址泄露——重要信息
  • Adobe 账号泄露无任何损失——普通/无所谓信息

然后分别对应不同的手机号码、邮件地址、不同的密码层级算法，做到基本上有3套互联网面具！

PS：

  1. 手机号码可以用「阿里小号」做到一机多号
  2. 电子邮件尽量使用「Google」、「Outlook」、「临时邮件」
  3. 密码不建议使用 「LastPass」 等工具记录，一旦被破解就……，而且此类事件在线密码管理器LastPass遭入侵已经发生

最小化原则 按照最小化原则，尽量少暴露个人信息，在信息上传之前请在内心问自己以下问题：

  • 网站数据能否注销
  • 这个数据是否一定要上传、此类服务是否有可替代服务
  • 数据能否使用虚拟的信息，如淘宝收件地址可使用 「易生 阿里小号号码 菜鸟驿站地址或不具体到单元楼层」
  • 数据是否做了相关的加密或水印（尤其是证件类）

追加一句，如果你涉及危险的网络行为请设计一套独立的网络账号（虚拟人，具体包含邮箱、IM、网站用户名）,并做相关的物理分割（不记名手机号码、物理电脑等）

其他细节

重要信息类
  • 上传照片时注意是否已消除GPS、拍摄时间等EXIF信息，证件类加「水印」
  • 网上绑定信用卡时用虚拟信用卡。信用卡不用时可以在手机银行中锁卡
  • 网购时最好是使用一张低额度的信用卡绑定快捷支付，其他全部走网银
  • 保护好银行卡的重要信息-「卡号、有效期、CVV 码」
  • 涉及到 QQ、微信等的朋友、亲人向你借钱、充值之类事宜，电话过去确认，要记住互联网那头不一定是一个人或你以为的那个人
相关操作类
  • 心理建设：不用相信「天下掉馅饼」的传说，高明的骗子一定会让你头脑发热，此时你的人性弱点正在被引导被利用，去洗个脸，冷静下，找个可信任的人说几句话，你就不会掉进去了
  • 将网上发布过的个人信息删除，如果不能删除的联系网站管理员删除
  • 重要账号开启两步验证，特别重要的账号可以用物理密钥
  • 云端同步的资料请注意判定其泄密的可能性以及后果，注意加密，具体可以访问Windows 教程 3……以防万一的网络云备份系统
  • 如果你是重要人物，记得给你的电脑 BIOS 加个密、硬盘加个密、关键文件放到 TrueCrypt 里，发邮件用 GPG 加密
  • 即使是官方短信也不一定可信，因为基站可伪造，百度「伪基站」看看就知道了，这条切记切记
  • 离开熟悉环境关闭手机 WiFi，防止自动连上不安全 WiFi。不得不用陌生 WiFi 时使用靠谱的 VPN，或者不登陆账户仅仅浏览，或者只使用全程 HTTPS 加密的服务
  • 用公用电脑或者别人电脑上网时，如果登陆账户，不保存密码，用完后退出账户，删除历史、cookies等，或者直接用隐私模式。
  • 去不知名的酒店入住时注意是否有摄像头偷拍，具体检查方法可参考知乎链接
  • 关键文件不要用迅雷下载，从原始地址下载，并且对比Hash值
  • 不要随意添加没有开源的浏览器插件，安装插件时检查权限
  • 笔记本电脑的摄像头用胶布粘住
